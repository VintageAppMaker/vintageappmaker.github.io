1. HTML 구조가 필요한 이유

HTML은 웹사이트의 '뼈대'입니다. 브라우저는 작성된 HTML 코드를 읽어 DOM(Document Object Model)이라는 트리 구조로 변환합니다.

Key Role

JavaScript는 이 DOM을 통해 특정 요소를 찾고, 내용을 바꾸거나 스타일을 변경하는 등 웹페이지를 살아 움직이게 만듭니다.

브라우저 시각화

Basic Hierarchy

<!DOCTYPE html> // 선언
<html> // 루트
  <head>
    // 메타데이터 영역
  </head>
  <body>
    // 콘텐츠 영역
  </body>
</html>
HEAD: 문서의 정보(메타데이터, 제목, 외부 파일 연결)를 담는 브레인 역할
BODY: 실제 사용자 화면에 보이는 콘텐츠를 담는 몸체 역할

3. 주요 Tag 요약 및 예제

Frontend 개발 시 빈번하게 사용되는 필수 태그

example.html 
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>Frontend 학습 페이지</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <!-- 레이아웃 및 텍스트 -->
    <div id="container">
        <h1>자바스크립트 학습</h1>
        <p class="description">HTML 제어는 JS의 핵심입니다.</p>
    </div>

    <!-- 목록 태그 -->
    <ul id="learning-list">
        <li>DOM 제어</li>
        <li>이벤트 처리</li>
    </ul>

    <!-- 폼 및 입력 -->
    <form id="loginForm">
        <input type="text" placeholder="아이디 입력">
        <button>확인</button>
    </form>
</body>
</html>

상세 기능 가이드

Meta & Head

UTF-8 인코딩 및 모바일 뷰포트 설정, SEO 최적화를 위한 Title 정보 포함.

Container Tags

div, h1~h3 등 레이아웃의 구획을 나누고 콘텐츠의 위계(Hierarchy)를 설정.

Input & Interaction

form, input, button을 사용하여 사용자와 데이터를 주고받는 통로 역할.

Pro Tip:

"script 태그는 보통 DOM 생성이 완료된 후 실행되도록 body 끝부분에 위치시키는 것이 관례입니다."

4. 핵심 태그 사전

프론트엔드 실무에서 놓치면 안 될 기능들

태그명 주요 용도 특이사항
<div> 콘텐츠 구획화 (Container) 가장 범용적인 레이아웃 도구
<ul> / <li> 순서 없는 리스트 메뉴바, 카드 리스트 구현 필수
<template> JS 동적 생성용 틀 화면엔 안보이지만 JS에서 복사해 사용
<meta> 문서 메타데이터 설정 인코딩(UTF-8), 뷰포트, SEO 담당
