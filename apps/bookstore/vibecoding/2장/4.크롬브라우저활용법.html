Web Debugging Essential

크롬 개발자 도구 완벽 가이드

웹페이지의 내부 구조를 실시간으로 확인하고, 에러를 추적하며 성능을 최적화하는 마스터 클래스입니다.

1

개발자 도구 시작하기

웹페이지가 의도대로 작동하지 않을 때 내부 코드(HTML, CSS, JS)를 실시간으로 확인하고 수정해보기 위해 반드시 필요한 도구입니다.

실행 단축키

Win / Linux F12 / Ctrl+Shift+I
Mac OS Cmd + Opt + I
브라우저 화면
연결
Elements HTML/CSS 뼈대 확인
Console 로그 및 코드 실행
Sources JS 중단점 디버깅
Network 서버 통신 데이터

2. HTML/CSS 디버깅 (Elements)

소스 파일을 직접 고치기 전, 브라우저에서 실시간으로 화면 구성을 테스트합니다.

  • ✓ 검사 모드: 상단 화살표 아이콘 클릭 후 화면 요소를 선택하여 코드로 즉시 이동
  • ✓ 텍스트 수정: 태그 우클릭 → Edit as HTML로 내용 변경 (예: Gmail → 쥐메일)
  • ✓ 스타일 조정: Styles 창에서 수치 조절 및 체크박스 제어로 디자인 실험

4. API 통신 (Network)

서버와 주고받는 데이터의 성공 여부를 파악합니다.

1. 필터 설정Fetch/XHR
2. 동작 발생버튼 클릭 등
3. 보낸 값Payload 확인
4. 받은 값Response 확인

3. 자바스크립트(JS) 디버깅

Console 활용

`Console` 탭에 로그를 출력하여 변수 값을 즉시 확인합니다.

let name = "Ninja";
console.log(name); 
// 콘솔창에 Ninja가 출력되는지 확인

* 팁: Ctrl + L을 누르면 콘솔을 비울 수 있습니다.

Sources 활용 (중단점)

Step 1

파일 목록에서 디버깅할 .js 파일 선택

Step 2

Breakpoint: 줄 번호를 클릭해 실행 일시정지

Step 3

Step Over: 버튼을 눌러 코드를 한 줄씩 실행하며 변수 변화 추적

5. 수정 내용 로컬에 반영 (Overrides)

폴더 지정

Sources → Overrides →
+ Select folder 선택 후 보안 허용

수정 및 저장

코드를 자유롭게 수정하고
Ctrl + S 로 저장

지속성 유지

새로고침(F5)을 해도
수정 사항이 사라지지 않고 유지

요약 가이드 ⚡️

UI가 이상할 때

Elements 탭

화살표 아이콘으로 클릭 확인

값이 궁금할 때

Console 탭

console.log 로 확인

코드가 꼬였을 때

Sources 탭

중단점 설정 후 한 줄씩 실행

데이터가 안 올 때

Network 탭

빨간색 에러 로그 확인
