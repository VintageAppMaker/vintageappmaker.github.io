01 바이브 코딩의 본질

바이브 코딩은 AI에게 정답을 구걸하는 것이 아닙니다. 명확한 문법적 지시(Command)를 통해 즉각적인 결과물을 도출하는 프로토타이핑 중심의 기법입니다.

JSON & LocalStorage

03 프롬프트 구조 (The 5 Blocks)

Block 01

역할 (Role)

AI의 페르소나를 정의하여 군더더기 없는 답변 유도

Block 02

목적 (Goal)

최종 결과물을 한 문장으로 명확히 선언

Block 03

제약 (Constraints)

환경, 라이브러리, 분량 등의 기술적 경계 설정

Block 04

형식 (Format)

코드블록만 출력할지, 마크다운을 쓸지 결정

Block 05

지침 (Behavior)

코딩 스타일, 예외 처리, 디버깅 로그 등 명시

가장 기초적인 상태 관리 프로토타입

오늘 할 일 3개 체크

UI 상호작용 및 스타일 제어

너는 설계 설명 없이 바로 화면을 만드는 코딩 파트너다. HTML과 JavaScript로 체크박스 3개가 있는 오늘 할 일 화면을 만든다. 체크하면 취소선이 그어진다.

URL 파라미터 로더

동적 데이터 렌더링

너는 빠르게 결과를 보여주는 HTML/JS 파트너다. index.html?title=약속 형식의 URL 파라미터를 읽어 화면에 출력하는 페이지를 만든다.

localStorage 메모장

브라우저 내 데이터 저장

HTML과 JavaScript로 메모 입력 화면을 만든다. 저장 버튼 클릭 시 localStorage에 저장하며 새로고침해도 유지된다.

프린트 전용 문서

특수 목적용 화면 설계

HTML/JS로 제목/내용 입력 시 프린트용 한 페이지 문서를 보여준다. print 시 버튼은 숨긴다.

랜덤 문장 표시기

데이터 배열 활용 로직
