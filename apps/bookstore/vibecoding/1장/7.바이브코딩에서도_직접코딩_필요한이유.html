왜 코딩은 반드시
직접 손으로 해야 할까?

"눈으로만 보고 이해하면 되는 것 아닌가요?"라는 질문에 대한 명쾌한 해답과 실무 경험 기반의 학습 가이드를 제안합니다.

01

코딩을 반드시 해야 하는 이유

이해와 구현의 격차

자바스크립트의 var 호이스팅이나 스코프 문제는 눈으로 볼 땐 쉽지만, 직접 짜보지 않으면 내가 언제 실수를 저지를지 알 수 없습니다.

AI 도구의 제어권

최근 유행하는 '바이브 코딩'도 원리를 모르면 AI 결과의 정오를 검증할 수 없습니다. "AI는 의도가 아니라 명세를 따른다"는 점을 기억하세요.

컴퓨팅 사고력

문제를 쪼개고(분해), 핵심만 남기는(추상화) 과정은 직접 타이핑하며 디버깅하는 고통의 시간 속에서 비로소 내 것이 됩니다.

03

간단한 예제: '오늘 할 일' 리스트

구현 로직 (작업 명세)

  • 1 <input> 창에서 글자를 입력받는다.
  • 2 '추가' 버튼 클릭 시 배열에 데이터를 담는다.
  • 3 화면의 <ul> 태그 안에 리스트를 출력한다.
  • 4 localStorage에 저장해 영속성을 유지한다.

💡 직접 해야만 배우는 것들

  • DOM 제어의 미묘한 디테일 (value vs text)
  • 사용자 예외 상황 대응 (빈 값 처리 등)
  • 코드가 화면을 바꾸는 성취감의 경험
Javascript
// 버튼 클릭 시 실행될 함수
function addTodo() {
  const input = document.getElementById('todoInput');
  const text = input.value; // 1. 입력값 받기

  // 예외 처리: 직접 해봐야 중요성을 깨닫는 구간
  if(text === "") {
    return alert("내용을 입력하세요!"); 
  }

  const li = document.createElement('li'); // 2. 요소 생성
  li.textContent = text;
  
  // 3. 화면 표시
  document.getElementById('todoList').appendChild(li); 
  input.value = ""; // 입력창 비우기
}
