데이터 타입과 형변환의 이해

컴퓨터는 사람과 다르게 숫자를 이해합니다. 데이터의 종류를 정확히 파악하고 올바르게 변환하는 것은 버그 없는 코딩의 첫걸음입니다.

💡 데이터 타입이 왜 필요할까요?

컴퓨터는 '숫자 5'와 '문자 "5"'를 완전히 다르게 인식합니다.

예상치 못한 결과:

5 + 5 = 10 (정상)
"5" + 5 = "55" (당황!)

이것을 방지하기 위해 데이터를 알맞은 상자에 담고, 필요할 때 상자의 종류를 바꾸는 방법을 알아야 합니다.

📦 Data Box 상자에 따라 결과가 달라집니다

데이터 타입 분류도

기본 타입 (Primitive)

값을 상자에 직접 담습니다

1
Number 1, -3, 0.14

숫자 형태의 데이터

String "Hello", 'Hi'

텍스트 형태의 데이터

Boolean true, false

참과 거짓 논리 데이터

2

참조 타입 (Object)

여러 값을 묶어서 관리합니다

Object { key: 'value' }

키-값 쌍의 집합

Array [1, 2, 3]

순서가 있는 데이터 목록

형변환: 데이터의 '통번역'

⚡ 암시적 형변환

자바스크립트가 똑똑한 척하며 스스로 타입을 바꿉니다. 편리하지만 의도치 않은 버그의 주요 원인이 됩니다.

"5" + 5 // → "55"

🛡️ 명시적 형변환

개발자가 직접 명령어를 사용해 타입을 바꿉니다. 예측 가능하고 안전한 코드 작성을 위해 권장됩니다.

Number("5") + 5 // → 10

실전 예제로 이해하기

1. 동적 데이터 타입 확인

Dynamic Type
// 처음엔 문자열을 담은 상자 let value = "안녕하세요"; console.log(typeof value); // "string" // 숫자를 넣으면 상자의 용도가 자동으로 바뀜 value = 100; console.log(typeof value); // "number"

자바스크립트는 상자의 용도를 수시로 바꿀 수 있는 유연함을 가졌지만, 나중에 무엇이 들었는지 헷갈릴 수 있으니 주의해야 합니다.

2. 명시적 형변환의 주요 함수들

Recommended
// 1. 숫자로 바꾸기 Number("2"); // 숫자 2 parseInt("142"); // 정수 142 // 2. 문자로 바꾸기 String(100); // 문자열 "100" // 3. 논리값으로 바꾸기 (0은 거짓, 그 외는 참) Boolean(0); // false Boolean(1); // true
