Asynchronous Programming

비동기 프로그래밍은 자바스크립트의 핵심입니다.
일상의 행동과 비교하여 그 원리를 쉽고 명확하게 파헤쳐 봅니다.

동기 (Synchronous)

음식이 나올 때까지 카운터 앞에서 기다리는 방식입니다. 앞사람의 작업이 끝날 때까지 전체 흐름이 멈춥니다.

"매우 비효율적이며 사용자 경험을 저해합니다."

비동기 (Asynchronous)

주문 후 진동벨을 받고 다른 일을 하며 기다리는 방식입니다. 데이터 로딩 중에도 스크롤이나 버튼 클릭이 가능합니다.

"현대 웹 애플리케이션의 필수 요소입니다."

비동기 작업의 상태 변화 (Promise)

⏳
Pending 대기 (주문 완료)
?
✅

Fulfilled

성공 (음식 나옴 → resolve)

❌

Rejected

실패 (재료 소진 → reject)

JS

실전 예제 (async/await)

// 1. 데이터를 가져오는 "약속" 함수 (2초 대기 시뮬레이션)
function 주문하기() {
  return new Promise((resolve, reject) => {
    console.log("1. 주문을 받았습니다. (요리 시작)");
    
    setTimeout(() => {
      const 성공 = true; // 요리 성공 여부
      if (성공) {
        resolve("🍕 피자"); // 성공 시 결과값 전달
      } else {
        reject("재료가 다 떨어졌어요."); // 실패 시 이유 전달
      }
    }, 2000); // 2초 뒤 실행
  });
}

// 2. 비동기 작업을 처리하는 함수
async function 점심식사() {
  console.log("--- 점심 시간 시작 ---");
  
  try {
    // await은 "결과가 나올 때까지 이 함수 안에서만 기다려라"라는 뜻입니다.
    const 음식 = await 주문하기(); 
    console.log(`3. 드디어 ${음식}가 나왔습니다! 맛있게 먹습니다.`);
  } catch (에러) {
    console.log(`X. 문제가 발생했어요: ${에러}`);
  }

  console.log("--- 점심 시간 종료 ---");
}

점심식사();
console.log("2. 요리가 준비되는 동안 다른 일을 합니다. (뉴스 보기)");

📝 실행 순서 분석

  1. 1

    점심 시간 시작 및 요리 시작 로그 출력

  2. 2

    "뉴스 보기" 작업이 즉시 실행 (전체 프로그램이 멈추지 않음!)

  3. 3

    2초 후, 요리가 완성되면 피자 먹기와 점심 시간 종료 출력

핵심 요약

  • Promise: 미래의 결과를 약속
  • Await: 작업 완료를 비동기적으로 대기
  • UX: 끊김 없는 사용자 경험 제공
