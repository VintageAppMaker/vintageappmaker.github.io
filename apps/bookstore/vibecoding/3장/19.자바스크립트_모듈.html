1

모듈(Module)이란 무엇인가?

개념 설명

모듈은 커다란 레고 성을 만들기 위한 '개별 레고 블록'과 같습니다. 하나의 거대한 자바스크립트 파일에 모든 코드를 작성하면 관리가 불가능해집니다. 이를 기능별로 쪼개어 독립된 파일로 만든 것이 바로 모듈입니다.

왜 필요한가?

  • 1
    코드의 재사용성 한 번 만든 기능을 다른 프로젝트에서도 쉽게 가져다 쓸 수 있습니다.
  • 2
    유지보수 용이 특정 기능에 문제가 생기면 해당 모듈 파일만 수정하면 됩니다.
  • 3
    네임스페이스 충돌 방지 서로 다른 파일에서 같은 이름의 변수를 사용해도 충돌하지 않습니다.
2

ES5 vs ES6 모듈 방식 비교

구분 ES5 (CommonJS 방식) ES6 (ESM 방식)
도입 시기 2009년 표준 (Node.js 기본 방식) 2015년 표준 (현대 프론트엔드 표준)
내보내기 키워드 module.exports / exports export / export default
가져오기 키워드 require() import
특징 실행 시점에 불러옴 (동적) 컴파일 시점에 불러옴 (정적, 성능 우수)
3

쉬운 예제로 보는 문법 비교

[예제 1] ES5 방식 (과거 방식)

더하기 기능을 가진 수학 도구함을 만든다고 가정해 봅시다.

math.js (내보내는 파일) 
// math.js
function add(a, b) {
  return a + b;
}

module.exports = add; // 도구함에서 add 함수를 밖으로 꺼내줌
app.js (가져오는 파일) 
// app.js
const add = require('./math.js'); // 꺼내준 add 함수를 불러옴

console.log(add(5, 3)); // 출력: 8
[예제 2] ES6 방식 (현대 표준 - import/export)

1) Named Export

여러 개의 기능을 각각의 이름으로 내보낼 때 사용합니다.

Multiple
math.js 
// math.js
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;
app.js 
// app.js
import { add, subtract } from './math.js';

console.log(add(5, 3));      // 8
console.log(subtract(5, 3)); // 2

2) Default Export

핵심 기능 하나만 내보낼 때 사용합니다.

Single
message.js 
// message.js
export default function greet(name) {
  return `안녕하세요, ${name}님!`;
}
app.js 
// app.js
// 이름을 자유롭게 지정 가능
import myGreet from './message.js'; 

console.log(myGreet('철수')); // 안녕하세요, 철수님!
4

왜 ES6 방식을 써야 할까요?

가독성

import, export라는 단어가 직관적이어서 코드가 무엇을 하는지 한눈에 알 수 있습니다.

성능 최적화

사용하지 않는 코드를 자동으로 제거(Tree Shaking)하여 파일 용량을 줄이고 속도를 높입니다.

브라우저 지원

최신 브라우저들은 별도의 설정 없이도 ES6 모듈을 직접 이해하고 처리할 수 있습니다.
