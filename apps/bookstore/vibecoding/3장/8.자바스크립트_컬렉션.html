Data Collections

자바스크립트에서 데이터를 다룰 때 가장 중요한 '컬렉션'에 대해 알아보겠습니다. 상황에 맞는 바구니를 선택하는 것이 고수의 지름길입니다.

어떤 바구니를 써야 할까요?

Array (배열)

데이터를 순서대로 나열하고 싶을 때 사용하세요.

Set (셋)

중복 없는 유일한 값들의 집합이 필요할 때 좋습니다.

Map (맵)

키와 값의 쌍으로 데이터를 빠르게 찾을 때 유리합니다.

한눈에 보는 핵심 요약

데이터 구조의 특징과 비유를 통해 쉽게 이해해 보세요.

타입 핵심 특징 비유
Array (배열) 순서가 있고, 중복을 허용함 번호표가 있는 대기 줄
Set (셋) 중복을 절대 허용하지 않음 중복 입장이 안 되는 파티장
Map (맵) 키(Key)와 값(Value)이 한 쌍임 사물함 번호와 내용물
1
ARRAY

순서가 중요한 데이터 바구니

가장 많이 쓰이는 컬렉션입니다. 넣은 순서대로 번호(인덱스)가 매겨집니다. push를 통해 데이터를 추가하고 인덱스로 접근할 수 있습니다. 

// 배열 생성
const fruits = ["사과", "바나나"];

// 데이터 추가
fruits.push("포도");
fruits.push("사과"); // 중복 데이터 추가 가능

console.log(fruits); 
// 출력: ["사과", "바나나", "포도", "사과"]

// 특정 위치의 데이터 가져오기 (0번부터 시작)
console.log(fruits[1]); // 출력: "바나나"

// Set 생성
const lotto = new Set();

// 데이터 추가
lotto.add(7);
lotto.add(21);
lotto.add(7); // 이미 있는 7은 무시됨

console.log(lotto); // 출력: Set(2) { 7, 21 }
console.log(lotto.size); // 출력: 2

// 데이터가 있는지 확인
console.log(lotto.has(7)); // 출력: true
2
SET

중복은 거부하는 바구니

유일한 값들만 모으고 싶을 때 사용합니다. 똑같은 데이터를 여러 번 add해도 하나만 남습니다. 데이터의 존재 여부를 확인할 때 속도가 매우 빠릅니다.

3
MAP

이름표가 붙은 바구니

데이터를 저장할 때 별칭(Key)을 붙여서 저장합니다. 일반 객체보다 데이터 추가/삭제가 빈번한 경우 성능이 더 우수합니다. 

// Map 생성
const userMap = new Map();

// 키-값 쌍으로 저장 (set 메서드)
userMap.set("id", "vintage_maker");
userMap.set("level", 5);

// 데이터 가져오기 (get 메서드)
console.log(userMap.get("id")); // 출력: "vintage_maker"

// 데이터 존재 확인
console.log(userMap.has("level")); // 출력: true

// 데이터 삭제
userMap.delete("level");

마지막 핵심 정리

  • 데이터를 순서대로 나열하려면? Array
  • 중복을 제거하고 관리하려면? Set
  • 이름표(Key)로 관리하려면? Map
