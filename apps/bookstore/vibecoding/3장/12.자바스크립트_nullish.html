Nullish Coalescing
연산자 (??) 가이드

데이터가 비어있을 때 안전하게 기본값을 지정하는 현대적인 방법.
기존의 논리 연산자(OR)가 가진 한계를 완벽하게 보완합니다.

1. 필요한 이유

서버 통신이나 사용자 입력 중 값이 null 또는 undefined인 경우가 많습니다. 이러한 상황에서 오류를 방지하고 화면이 깨지지 않도록 하는 "안전장치"가 필요합니다.

💡

값이 없으면 이 기본값을 대신 써라! 라고 명시적으로 지정하여 프로그램의 안정성을 높입니다.

2. 핵심 로직 도식

[데이터 A] ?? [기본값 B]
A가 null 또는
undefined인가?
Logic
YES
[기본값 B] 사용
NO
[데이터 A] 사용

# 상태 구분

Nullish 값 (정말로 비어있음)

  • null
  • undefined

일반 Falsy 값 (정상 데이터 취급)

  • 0 (숫자 영)
  • "" (빈 문자열)
  • false (거짓)

3. 실무 코드 예제

Example 01: 데이터가 존재하지 않을 때

let value = null;
let result = value ?? "기본값";

console.log(result); 
// 출력: "기본값"

Example 02: 0이나 빈 문자열이 데이터일 때

let score = 0;
let finalScore = score ?? 100;

console.log(finalScore); 
// 출력: 0 
// (0은 nullish가 아니므로 그대로 사용됨)

4. 왜 ?? 인가? (OR 연산자 || 와의 차이)

연산자 기본값 선택 기준 숫자 0의 처리 실무 적합성
|| (OR) Falsy한 모든 값 기본값으로 덮어씀 데이터 손실 위험
?? (Nullish) null, undefined만 0을 데이터로 보존 가장 안전함 (권장)

💡 결론

교재 34~35페이지의 설명처럼, 게임 점수가 0점일 때 ??를 사용하면 0을 정상적으로 표시할 수 있습니다. 반면 ||를 사용하면 0을 "데이터 없음"으로 오해해 기본값으로 덮어씌우는 오류가 발생합니다.
