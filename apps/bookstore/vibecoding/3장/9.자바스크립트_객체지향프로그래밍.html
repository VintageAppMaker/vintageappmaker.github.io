객체 지향 프로그래밍 (OOP)

인간 세상의 사물을 소프트웨어로 모델링하는 현대 프로그래밍의 핵심 철학과 자바스크립트의 프로토타입 시스템을 이해합니다.

1. 필요한 이유

코드 재사용

똑같은 기능을 매번 새로 만들지 않고, '틀'을 하나 만들어 필요할 때마다 재사용하여 생산성을 극대화합니다.

유지보수 용이

레고 블록처럼 기능을 분리해 두면, 특정 부분에 문제가 생겼을 때 전체를 건드리지 않고 해당 블록만 수정하면 됩니다.

복잡성 해결

프로그램이 커질수록 데이터를 구조적으로 관리해야 엉키지 않습니다. 데이터와 기능을 하나로 묶어 관리합니다.

2. 핵심 개념 도식화

붕어빵 틀 모델 (Class vs Object)

📋
Class (설계도)
이름, 재료, 굽는 법 정의
🐟
Object (실제 객체)
메모리에 생성된 실제 결과물

상속 관계 도식화

부모 (Prototype / Class)
[먹이, 숨쉬기()]
상속 (Inheritance)
▼
자식 (Instance / Object)
[강아지, 멍멍!()]

자바스크립트 객체는 __proto__ 링크를 통해 조상의 기능을 빌려 씁니다.

3. 현대적 방식: ES6 Class

example.js
// 1. 설계도(Class) 만들기 class Animal { constructor(name) { this.name = name; // 데이터(필드) 정의 } cry() { return `${this.name}이(가) 소리를 냅니다.`; // 기능(메소드) 정의 } } // 2. 상속받아 새로운 틀 만들기 class Dog extends Animal { bark() { return "멍멍!"; } } // 3. 실제 객체(Instance) 생성 const myDog = new Dog("시고르자브종"); console.log(myDog.cry()); // "시고르자브종이(가) 소리를 냅니다." (부모 기능 사용) console.log(myDog.bark()); // "멍멍!" (본인 기능 사용)

4. 예제 상세 설명

  • 1

    class Animal

    동물의 공통적인 특징을 담은 상위 설계도입니다.

  • 2

    constructor(name)

    객체가 생성될 때 가장 먼저 실행되는 생성자 함수로, 초기 데이터를 설정합니다.

  • 3

    extends (상속)

    Dog가 Animal의 모든 능력을 물려받습니다. cry() 기능을 재구현할 필요가 없습니다.

프로토타입 체인

자바스크립트는 객체에 기능이 없으면 "너 이거 없니? 그럼 부모한테 있나 볼게" 하고 링크를 따라 올라갑니다. 이것이 프로토타입 체인의 핵심입니다.
