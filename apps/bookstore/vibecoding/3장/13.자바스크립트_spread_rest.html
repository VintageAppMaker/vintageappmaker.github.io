왜 필요한가요?

우리가 일상에서 물건을 다룰 때, "가방에 있는 물건을 하나씩 꺼내서 펼쳐놓아야 할 때"가 있고, 반대로 "남은 물건들을 한꺼번에 모아서 상자에 담아야 할 때"가 있죠?

  • 복사 및 병합: 기존 데이터를 건드리지 않고 똑같은 복사본을 만들거나 여러 데이터를 하나로 합칠 때 코드가 훨씬 간결해집니다.

  • 유연한 대응: 함수에 데이터가 몇 개 들어올지 모르는 상황에서 "나머지 전부"를 똑똑하게 처리할 수 있게 해줍니다.

실무 활용 예제

1

Spread: 배열 합치기

...snacks라고 쓰면 ["새우깡", "감자칩"]이라는 봉투가 터지면서 내용물이 낱개로 튀어나옵니다. 만약 ... 없이 넣었다면 배열 안에 또 배열이 들어가는 복잡한 구조가 되었을 거예요.

// 두 배열을 합쳐서 새로운 장바구니 만들기 const snacks = ["새우깡", "감자칩"]; const drinks = ["콜라", "사이다"]; const cart = [...snacks, ...drinks, "초콜릿"]; console.log(cart); // 결과: ["새우깡", "감자칩", "콜라", "사이다", "초콜릿"]
2

Rest: 나머지 다 가져오기

...others는 앞서 지정한 leader를 제외하고 "남은(Rest) 데이터"가 몇 개든 상관없이 전부 모아서 하나의 새로운 배열로 만들어줍니다. 일일이 이름을 붙일 필요가 없죠.

// 첫 번째는 반장, 나머지는 일반 학생으로 구분하기 const students = ["민수", "철수", "영희", "길동"]; const [leader, ...others] = students; console.log(leader); // 결과: "민수" console.log(others); // 결과: ["철수", "영희", "길동"]

정리하자면!

데이터를 낱개로 풀어서 쓰고 싶을 땐 Spread,
남은 것들을 한데 묶어서 처리하고 싶을 땐 Rest를 기억하세요!
