논리 OR 연산자 (||) 이해하기

데이터가 비어있어도 프로그램이 멈추지 않도록 도와주는 자바스크립트의 필수 연산자, OR 연산자의 작동 원리와 활용법을 알아봅니다.

1. 필요한 이유

사용자로부터 값을 입력받지 못하거나, API 응답 데이터가 비어 있는 경우(null, undefined 등) 프로그램은 예기치 못한 에러를 발생시킬 수 있습니다.

데이터가 비어있어도 프로그램이 멈추지 않고 '기본값(Default value)'을 가지고 계속 작동하게 만들기 위해 필수적입니다.

JavaScript의 Falsy 값

아래 값들은 JS에서 '거짓'으로 간주되어 OR 연산 시 오른쪽 값을 선택하게 만듭니다.

null
undefined
0
"" (빈 문자열)
NaN
false

2. 핵심 로직 도식화

Operand A
[ 왼쪽 값 ]
||
OR 연산자
Operand B
[ 오른쪽 값 ]
플랜 B / 기본값

왼쪽이 참(Truthy)인 경우

왼쪽 값을 즉시 반환하며, 오른쪽은 무시됩니다.

왼쪽이 거짓(Falsy)인 경우

오른쪽 값(기본값)을 선택하여 반환합니다.

example.js 

// 1. 데이터가 비어있는 경우 (null)
let inputName = null;
let userName = inputName || "익명사용자";
console.log(userName); // 결과: "익명사용자"

// 2. 데이터가 있는 경우
let inputName2 = "홍길동";
let userName2 = inputName2 || "익명사용자";
console.log(userName2); // 결과: "홍길동"

// 3. 숫자 0인 경우 (0은 Falsy 값임)
let score = 0;
let finalScore = score || 100;
console.log(finalScore); // 결과: 100

🔍 예제 상세 설명

  • 01.

    null은 거짓으로 판단되어 우측의 기본값인 "익명사용자"를 선택합니다.

  • 02.

    이미 값이 있는 경우 연산자는 즉시 해당 값을 채택하며 우측은 실행되지 않습니다.

⚠️ 주의: 숫자 0

JavaScript에서 0은 Falsy 값입니다. 실제 점수가 0점인 경우에도 기본값이 들어갈 수 있으니 주의하세요.

Solution

이런 경우에는 nullish 병합 연산자 (??)를 사용하는 것이 좋습니다.
