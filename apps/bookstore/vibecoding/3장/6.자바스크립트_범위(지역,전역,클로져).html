Scope & Closure

자바스크립트 개발자이자 강사가 들려주는 일상 비유 가이드.
복잡한 개념을 가장 쉽고 우아하게 이해해보세요.

1. 스코프 (Scope)

● 변수가 살 수 있는 "동네"

스코프는 변수나 함수가 어디까지 영향을 미칠 수 있는지, 즉 "접근 가능한 범위"를 말합니다. 모든 변수를 한곳에 두면 이름 충돌이 발생하므로 동네(영역)를 나누어 관리합니다.

Global 전역 스코프: 광장

누구나 접근 가능

Local 지역 스코프: 우리 집

집 안 식구만 접근 가능

Local Block 지역 스코프: 내 방

나만 접근 가능

JavaScript
let gVar = "광장의 분수"; // 전역 (Global)

function myHome() {
  let lVar = "우리 집 TV"; // 지역 (Local)
  
  console.log(gVar); // 접근 가능 (집에서 광장이 보임)
  console.log(lVar); // 접근 가능 (내 집 물건임)
}

myHome();
console.log(gVar); // 접근 가능
// console.log(lVar); // 에러! (광장에서 남의 집 TV 못 봄)

"안에서 밖은 볼 수 있지만, 밖에서 안은 들여다볼 수 없습니다."

2. 클로저 (Closure)

Closure Example
function createCounter() {
  let count = 0; // 외부 함수의 변수 (은닉)

  return function() { // 내부 함수 (클로저)
    count++; // 태어난 곳의 환경을 기억함
    return count;
  };
}

const myCounter = createCounter(); 

console.log(myCounter()); // 1
console.log(myCounter()); // 2
console.log(myCounter()); // 3

● "과거를 기억하는 함수"

클로저는 함수가 생성될 때 자신이 태어난 환경(스코프)을 "기억"했다가 나중에 호출될 때 그 기억을 꺼내 쓰는 기능입니다.

1
외부 함수 실행 종료
2
[내부 함수]가 외부 변수를 꽉 잡고 있음!
3
호출 시마다 저장된 기억을 꺼내 사용

결과: 외부에서는 count를 직접 건드릴 수 없지만, myCounter를 통해서는 안전하게 데이터를 유지/관리할 수 있습니다 (상태 보존 및 은닉).

요약 정리 (Summary)

개념 일상 비유 핵심 특징
전역 스코프 마을 광장 어디서든 누구나 접근 가능
지역 스코프 개인 주택 울타리({ }) 안에서만 접근 가능
클로저 도시락 가방 함수가 끝난 뒤에도 태어난 동네의 물건을 챙겨 다니는 것

"스코프는 어디까지 보이나?의 문제이고,
클로저는 환경을 어떻게 살려두나?의 지혜입니다."
