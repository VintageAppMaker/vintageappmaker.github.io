핵심 연산자 2가지

자바스크립트 개발의 숙련도를 결정짓는 !!(이중 NOT)과 ?.(옵셔널 체이닝)을 통해 더 깔끔하고 견고한 코드를 작성하는 방법을 알아봅니다.

🧪

1. !! 연산자

"확실한 참/거짓 판독기" - 어떤 값이든 명확한 Boolean 타입으로 변환하고 싶을 때 사용합니다. (매뉴얼 36p)

필요한 이유

Falsy(0, "", null 등)와 Truthy 값이 혼재된 환경에서 논리 구조를 단순화하고 타입의 모호함을 제거하기 위해 사용합니다.

Process Logic (도식)

원래 값
!
반대로 뒤집기
!
원래 논리로 복원
순수 Boolean
Code Implementation 
let value1 = "hello"; // 값이 들어있는 문자열
let value2 = "";      // 비어있는 문자열

console.log(!!value1); // true
console.log(!!value2); // false

* 설명: 첫 번째 !가 논리를 반전시키고, 두 번째 !가 다시 반전시켜 원래의 참/거짓 성질을 Boolean 객체로 확정합니다.

🛡️

2. ?. 연산자

"에러 방지 안전벨트" - 데이터 계층 구조를 탐색할 때 TypeError로 프로그램이 멈추는 것을 방지합니다. (매뉴얼 37p)

필요한 이유

서버에서 받아온 데이터가 불완전하거나 비어있을 때, 존재하지 않는 속성에 접근하면 발생하는 런타임 에러를 우아하게 처리합니다.

Safe Navigation Logic (도식)

데이터가 있을 때

다음 단계의 속성에 정상적으로 접근하여 값을 반환합니다.

데이터가 없을 때 (null/undefined)

즉시 실행을 멈추고 에러 없이 undefined를 반환합니다.

Code Implementation 
let hero = {
  id: "onePunch",
  profile: {
    name: "Saitama"
  }
};

// 정상 접근
console.log(hero?.id); // "onePunch"

// 존재하지 않는 속성 접근 (salary는 profile 안에 없음)
console.log(hero?.profile?.salary); // undefined (에러 안 남!)

* 설명: 복잡한 중첩 객체 탐색 시 수많은 if문을 대체하여 코드의 가독성을 획기적으로 개선합니다.

핵심 요약 정리 (매뉴얼 38p)

연산자 용도 핵심 특징
!! 불리언 변환 어떤 값이든 명확한 true/false로 바꿈
?. 안전한 탐색 에러 대신 undefined를 반환하여 프로그램 정지를 방지

버그를 줄이는 가장 쉬운 방법!
