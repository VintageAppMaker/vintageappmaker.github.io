JavaScript Essentials

자바스크립트 함수(Function)

함수는 특정 작업을 수행하기 위해 독립적으로 설계된 코드 묶음입니다.
마치 레고 블록처럼 조립하여 더 큰 프로그램을 만듭니다.

01 함수의 실행 메커니즘

1. 입력 (Parameter)

함수로 전달되는 값

2. 처리 (Logic)

데이터 가공 및 연산

3. 출력 (Return)

최종 결과값 반환

02 함수 정의의 세 가지 핵심 방식

Traditional

1. 함수 선언문 (Declaration)

가장 전통적이고 대중적인 방식입니다. "미리 등록해둔 레시피"와 같아서 이름만 부르면 어디서든 실행됩니다.

  • ✦ 호이스팅(Hoisting) 지원: 함수 선언 전 윗부분에서 호출해도 에러 없이 작동합니다. 
// 선언하기 전에도 호출 가능!
console.log(sayHi()); 

function sayHi() {
  return "안녕하세요! 일반 함수입니다.";
}
Logical

2. 함수 표현식 (Expression)

함수를 익명 상태로 만들어 변수에 담는 방식입니다. 데이터처럼 취급할 수 있어 유연성이 높습니다.

  • ✦ 순차적 실행: 호이스팅이 발생하지 않아 반드시 선언 후 사용해야 합니다. (코드의 안정성) 
// 변수에 함수를 대입
const subtract = function(a, b) {
  return a - b;
};

// 선언 이후에만 사용 가능
console.log(subtract(10, 3)); // 출력: 7
Modern

3. 화살표 함수 (Arrow)

ES6에서 도입된 "속성 요약 레시피"입니다. 간결한 문법 덕분에 현대 프론트엔드 개발(React 등)에서 가장 많이 쓰입니다.

  • ✦ 극강의 간결함: 한 줄일 경우 return과 {} 생략이 가능합니다. 
// 한 줄 요약 버전
const multiply = (a, b) => a * b;
console.log(multiply(2, 4)); // 8

// 여러 줄 로직 버전
const getMessage = (name) => {
  let greeting = "반가워요, " + name + "님!";
  return greeting;
};

03 💡 한눈에 비교하기

구분 문법 예시 호이스팅 주요 특징
함수 선언문 function name() {} 가능 가장 기본, 전역 어디서나 호출 가능
함수 표현식 const name = function() {} 불가능 변수처럼 사용, 논리적 흐름 중시
화살표 함수 const name = () => {} 불가능 간결한 문법, 현대 개발 표준 선호

현명한 개발자를 위한 제언

현대 자바스크립트 개발에서는 화살표 함수를 가장 많이 사용합니다. 특히 콜백 함수나 짧은 연산에서 압도적인 가독성을 자랑합니다. 다만, 코드의 실행 흐름을 예측 가능하게 유지하기 위해 함수를 먼저 정의하고 사용하는 표현식 방식의 습관을 들이는 것이 좋습니다.
