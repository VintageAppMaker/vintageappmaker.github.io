01 API가 필요한 이유

식당에서 손님이 주방의 요리 과정을 다 알 필요가 없는 것처럼, 프로그래밍에서도 외부 시스템의 복잡한 내부 코드를 모두 이해할 필요는 없습니다.

기상청의 날씨 정보나 구글 지도를 쓰고 싶을 때, 그들이 정해놓은 '주문 방식(API)'에 맞춰 요청만 하면 필요한 데이터를 즉시 받을 수 있습니다.

03 지금 바로 테스트하기

Chrome DevTools Console
1
실습 방법
  1. 01. 브라우저에서 F12를 눌러 개발자 도구를 엽니다.
  2. 02. 상단 탭에서 Console(콘솔)을 클릭합니다.
  3. 03. 오른쪽 코드를 복사해서 붙여넣고 Enter를 누르세요.
Tip
무료 테스트용 API를 활용해 실제 서버 통신을 경험할 수 있습니다.
// 무료 테스트용 API 주소에서 '1번 할 일' 정보를 가져오는 예제입니다.
fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => response.json()) // 응답을 JSON 형식으로 변환
  .then(data => {
    console.log("받아온 데이터:", data);
    alert("제목: " + data.title); // 알림창으로 제목 출력
  })
  .catch(error => console.error("통신 에러:", error));

04 코드 상세 설명

fetch('URL')

해당 주소(서버)로 데이터를 가지러 가라는 명령입니다. 기본적으로 정보를 요청하는 GET 방식이 사용됩니다.

.then(response => response.json())

서버 응답이 도착하면, 그 원시 데이터를 우리가 읽기 쉬운 JSON 객체로 변환하는 약속(Promise) 단계입니다.

.then(data => { ... })

변환이 완료된 실제 데이터(data)를 가공하는 구간입니다. 화면에 출력하거나 특정 로직을 처리하는 핵심 로직이 들어갑니다.

.catch(error => ...)

네트워크 장애, 잘못된 주소 등으로 통신이 실패했을 때를 대비한 안전망입니다. 에러 원인을 파악할 수 있게 도와줍니다.

결론: API의 가치

API를 활용하면 복잡한 데이터베이스를 직접 구축하지 않아도, 외부의 양질의 데이터를 내 웹사이트나 앱에 쉽게 가져와 사용할 수 있습니다. 이는 개발 속도를 획기적으로 높이고 더 강력한 서비스를 만드는 원동력이 됩니다.
